## 2026-05-23

### 修复 Moshi_VolumeCN.cs NullReferenceException
- **问题**: `DrawComponentParameters` 中 `EditorUtility.SetDirty(targetProfile ?? targetVolume?.sharedProfile)` 在自动检测场景 Volume 但未手动赋值 `targetProfile`/`targetVolume` 时，表达式结果为 null，导致 `ArgumentNullException`。
- **修复**: 新增 `activeWorkingProfile` 字段，在 `OnGUI` 中赋值（含自动检测逻辑），`DrawComponentParameters` 中改用该字段并加固 null 检查。

### 修复 Newtonsoft.Json 版本冲突
- **问题**: `com.unity.collab-proxy@1.7.1` 引用 Newtonsoft.Json 12.0，但项目通过 `com.unity.nuget.newtonsoft-json@3.2.0` 提供 13.0，导致 unityplastic.dll 加载失败。
- **修复**: 升级 `com.unity.collab-proxy` 从 `1.7.1` → `2.0.4`（兼容 Unity 2021.3，已适配 Newtonsoft 13.0）。同步更新 `packages-lock.json`。

### 修复 Moshi_VolumeCN 性能与子模块控制（第三版 - 最终）
- **彻底消除延迟**: 放弃手动反射方案，`DrawComponentParameters` 改为 `SerializedObject` + `SerializedProperty` 驱动。所有字段通过 `so.FindProperty` → `FindPropertyRelative("m_OverrideState"/"m_Value")` → `EditorGUILayout.PropertyField` 渲染。Unity 原生序列化系统处理变更检测（`hasModifiedProperties`），无需任何反射 GetValue/SetValue。
- **序列化属性名修正**: VolumeParameter 中 overrideState 序列化名为 `m_OverrideState`，value 为 `m_Value`（不是 C# 属性名），带 fallback 兼容。
- **原生拖拽支持**: `PropertyField(valueProp, GUIContent.none, true)` 原生支持所有 Unity 类型（float/int/Color/Vector/Texure/AnimationCurve/TextureCurve/Enum/Struct），拖拽、滑块全原生。
- **精确 Dirty 标记**: `DrawComponentParameters` 返回 `bool`，仅 `so.hasModifiedProperties == true` 时标记 Profile 为 dirty，避免无变化时的多余序列化。
- **优化自动检测**: `FindObjectOfType<Volume>()` 首次结果缓存到 `activeWorkingProfile`，后续帧不再重复查找。
- **ToggleLeft 布局**: 组件级和参数级均使用 `ToggleLeft(中文名)`，标签文字本身可点击，不再受 16px 限制。
- **即时反馈刷新**: 新增 `RepaintUnityInspector()` → 通过 `Resources.FindObjectsOfTypeAll` 找到所有 InspectorWindow + SceneView 并强制 Repaint。组件级 toggle 和参数级变更均直接调用（无 delayCall 包装），因为 `EditorWindow.Repaint()` 只是标记下帧重绘，在 OnGUI 中调用跨窗口 Repaint 安全。修改参数/开关后原生 Inspector 和场景视图均在同一帧内同步。
