# 批量重命名工具 (Moshi_BatchRename)

## 功能概述
增强版文件批量处理工具，支持对 Project 资源和 Hierarchy 对象进行批量重命名，以及批量替换文件内容。

## 打开方式
**菜单路径**: `工具 > Moshi > 批量重命名`

## 功能模块

### 1. Project 窗口标签页

#### 批量重命名文件
支持两种重命名模式：

**修改模式（基于原名）**:
- **查找内容**: 在原文件名中查找的文本
- **替换为**: 替换后的文本
- **文件抬头名**: 添加到文件名开头的前缀
- **文件后缀名**: 添加到文件名末尾的后缀
- **自动编号**: 启用后可设置起始值、步长、位数

**直接重命名模式**:
- **分隔符**: 各部分之间的连接符（默认 `_`）
- **抬头/中段/后缀**: 可分别启用并设置内容
- **编号位置**: 可选择放在最前、中间或最后

**文件来源**:
- **选中文件**: 处理 Project 中当前选中的文件
- **指定文件夹**: 处理指定文件夹中的所有文件（支持多文件夹、拖拽添加）

**其他选项**:
- **按名称排序**: 重命名前先按名称排序
- **文件类型过滤**: 只处理指定扩展名的文件（如 `.prefab;.mat`）

#### 批量替换文件内容
在文本类文件中批量查找替换内容。
- 支持的文件类型: `.txt`, `.json`, `.csv`, `.xml`, `.cs`, `.shader` 等

### 2. Hierarchy 面板标签页
用于批量重命名场景中的 GameObject。

**功能特点**:
- 支持与 Project 相同的修改模式和直接重命名模式
- **包含子对象**: 是否同时处理选中对象的所有子对象
- **支持撤销**: 可使用 Ctrl+Z 撤销操作

### 3. 重命名预览
执行重命名前可先生成预览，确认无误后再执行。

## 使用流程
1. 选择标签页（Project/Hierarchy）
2. 选择文件来源或对象
3. 配置重命名参数
4. 点击"生成重命名预览"查看效果
5. 确认无误后点击"执行重命名"

## 注意事项
1. 执行前建议先预览确认
2. Hierarchy 重命名支持撤销，Project 重命名不支持
3. 文件类型过滤使用分号或逗号分隔多个扩展名
