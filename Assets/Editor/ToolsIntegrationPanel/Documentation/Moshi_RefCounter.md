# 引用计数统计工具 (Moshi_RefCounter)

## 功能概述
专注于 Texture、Material、Mesh 和 Animation 资源的引用统计，帮助发现零引用的冗余资源，并提供资源依赖整合功能。

## 打开方式
**菜单路径**: `工具 > Moshi > 引用计数统计`

## 功能模块

### 1. 资源引用统计

#### 资源类型设置
- 包含 Texture 资源
- 包含 Material 资源
- 包含 Mesh 资源
- 包含 Animation 资源

#### 扫描范围设置
指定一个目标文件夹，只统计该文件夹内的资源引用情况。

#### 统计功能
- **统计选中资源引用**: 统计当前在 Project 中选中的资源
- **统计所有资源引用**: 统计目标文件夹内所有符合类型的资源
- **标记零引用资源**: 标记没有被任何其他资源引用的资源
- **导出清理报告**: 导出 CSV 格式的零引用资源报告

### 2. 零引用资源管理
- **全选/取消全选**: 批量选择零引用资源
- **删除选中**: 删除选中的零引用资源
- **一键删除全部**: 删除所有零引用资源
- **定位**: 在 Project 中定位资源
- **删除**: 单独删除某个资源

### 3. 资源依赖整合工具

用于将分散在不同文件夹的依赖资源整合到主文件夹中。

#### 配置项
- **主文件夹**: 目标整合文件夹
- **副文件夹**: 手动添加的额外扫描文件夹
- **白名单文件夹**: 不需要整合的依赖来源（如公共资源库）

#### 文件夹名称配置
可自定义各资源类型的目标子文件夹名称：
- Texture、Material、Mesh、Animation
- Prefab、Shader、Audio、ScriptableObject
- Timeline、Font、Video 等

#### 整合流程
1. 设置主文件夹
2. 配置白名单文件夹（可选）
3. 点击"扫描依赖"分析依赖关系
4. 查看扫描结果：
   - 外部依赖（红色）：需要整合的资源
   - 白名单依赖（白色）：不需要整合
   - 内部依赖（绿色）：已在主文件夹内
5. 点击"一键整合资源"执行整合

## 注意事项
1. 删除资源操作不可撤销，请谨慎操作
2. 建议在操作前备份项目
3. 资源整合会自动更新引用关系
4. 白名单设置会自动保存
5. Shader 资源不会被整合（避免破坏依赖）
