# Moshi 工具箱导出

## 简介

**工具箱导出** 工具可以一键打包所有或单独指定的 Moshi 工具为 ZIP 或 UnityPackage 格式，方便分享给其他开发者。

## 功能特性

- **两种导出格式**：ZIP（轻量分享）和 UnityPackage（Unity标准格式）
- **两种导出范围**：全部导出 / 单独导出
- **自动扫描**：自动收集编辑器工具和运行时脚本
- **.meta 开关**：可自由选择是否包含 .meta 文件
- **个人数据检测**：自动识别并标记个人配置数据
- **可选导出**：个人数据默认排除，可手动勾选一同导出
- **文件预览**：导出前可查看完整文件列表和大小统计
- **新工具自动识别**：新增工具按规范放置后自动纳入导出范围

## 使用方法

1. 打开工具：**工具 → Moshi → 工具箱导出**
2. 选择导出格式（ZIP 或 UnityPackage）
3. 选择导出范围（全部导出 或 单独导出）
4. 根据模式配置导出内容
5. 选择是否包含 .meta 文件
6. 点击 **导出打包** 按钮
7. 选择保存位置

## 导出范围

### 全部导出

打包所有 Moshi 工具，可分别勾选：
- **编辑器工具**（`Assets/Editor/ToolsIntegrationPanel/`）
- **运行时脚本**（`Assets/Scripts/MoshiScripts/`）
- **个人数据**（默认排除，可单独勾选）

### 单独导出

选择性导出一个或多个工具：
- 自动扫描所有 `Moshi_` 前缀的工具
- 自动识别工具的 Editor 和 Runtime 对应关系（如 `Moshi_PathTool` 同时有 Editor 和 Runtime 部分）
- 支持搜索过滤工具名称
- 全选 / 全不选快捷操作
- 可选是否包含框架共享文件（面板脚本、文档等）

**工具列表标签说明：**

| 标签 | 含义 |
|------|------|
| `E:5` | 该工具有 5 个 Editor 文件 |
| `R:3` | 该工具有 3 个 Runtime 文件 |

**自动命名规则：**
- 单个工具：`Moshi_PathTool_2026-03-26.zip`
- 多个工具：`MoshiTools_x3_2026-03-26.zip`

## 导出格式区别

| 特性 | ZIP | UnityPackage |
|------|-----|--------------|
| 适用场景 | 代码分享/备份 | Unity项目导入 |
| 导入方式 | 解压后合并到 Assets/ | Assets → Import Package |

## .meta 文件开关

两种格式和两种导出范围都可以独立控制是否包含 `.meta` 文件：
- **勾选**：保留 GUID 引用关系，Unity 导入时不会重新生成引用
- **不勾选**：体积更小，适合纯代码分享

> ⚠️ 使用 UnityPackage 格式时，建议勾选 .meta 以保持 GUID 引用。不勾选会在导出前弹出提醒。

## 个人数据说明

以下文件被识别为个人数据，默认不导出（仅全部导出模式显示）：

| 文件 | 说明 |
|------|------|
| `MyToolsConfig.asset` | 工具面板排列与快捷访问配置 |
| `Moshi_FavoritesConfig.asset` | 资源收藏夹列表 |
| `Moshi_ImgClassifier_Config.json` | 图片分类器学习数据 |

勾选后可一同导出，导出前会弹出二次确认。

## 注意事项

- 导入 UnityPackage 后脚本编译完成时，会通过 `[InitializeOnLoad]` 机制**自动同步**配置：
  - `MyToolsConfig.asset` **不存在** → 自动创建（全量生成）
  - `MyToolsConfig.asset` **已存在**（随包导入了个人配置）→ 自动同步：补充当前项目有但配置中缺失的工具，清理配置中已失效的条目，**保留原有排列顺序和快捷访问设置**
- 若自动同步未触发，可打开 `Tools > MoshiResourceTools` 面板，点击 **"配置刷新"** 按钮手动同步
- 不含 .meta 文件时，导入后 Unity 会重新生成 GUID（预制体引用可能断裂）
- 建议正式分享使用 UnityPackage 格式 + 勾选 .meta
- 单独导出时，框架共享文件开关默认开启，确保工具面板可用
