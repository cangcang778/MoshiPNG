# 资源清理 (Moshi_AssetClean)

## 功能概述
五合一资源清理工具，集成**空材质检测**、**残留清理**、**缩放修复**、**空粒子清理**、**贴图检查**功能。每个功能对应一个标签页，通过顶部按钮组切换。

## 打开方式
**菜单路径**: `工具 > Moshi > 资源清理`

---

## 共享扫描范围（前4个标签页通用）

前4个标签页（空材质检测、残留清理、缩放修复、空粒子清理）共享同一个扫描范围设置：

| 扫描范围 | 说明 |
|---------|------|
| **选中对象** | 扫描 Hierarchy 中当前选中的 GameObject（支持多选） |
| **当前场景** | 扫描当前活动场景中所有根物体及其子物体 |
| **指定预设** | 指定单个 Prefab 或文件夹，搜索其中所有 `.prefab` 文件 |

### 指定预设模式额外设置
- **预设目标**: 拖入单个 Prefab 文件
- **扫描文件夹**: 拖入 Project 中的文件夹
- **包含子文件夹**: 是否递归搜索子文件夹中的 Prefab（默认开启）

> 💡 **贴图检查**标签页使用独立的文件夹路径设置，不使用共享扫描范围。

## 标签页 1：空材质检测

### 功能说明
检测选中对象/场景/预制体中各类渲染器组件的材质引用，找出材质球为空（Missing 或未赋值）、材质球数组中存在空槽位的问题。

### 检测类型（6种渲染器，均可独立开关）

| 检测类型 | 检测对象 | 默认 |
|---------|---------|------|
| 粒子渲染器 | `ParticleSystemRenderer` 的 sharedMaterials | ✅ |
| 网格渲染器 | `MeshRenderer` 的 sharedMaterials（排除含 ParticleSystem 的物体） | ✅ |
| 蒙皮渲染器 | `SkinnedMeshRenderer` 的 sharedMaterials | ✅ |
| 精灵渲染器 | `SpriteRenderer` 的 sharedMaterials | ✅ |
| 线条/拖尾 | `LineRenderer` + `TrailRenderer` 的 sharedMaterials | ✅ |
| 粒子拖尾 | ParticleSystem 的 trails 模块启用时的 trailMaterial | ✅ |

### 操作按钮

| 按钮 | 功能 |
|------|------|
| **开始扫描** | 递归扫描所有目标物体及子物体，检测空材质 |
| **复制名称** | 将所有问题物体的名称（去重）复制到剪贴板 |
| **导出文件** | 弹出保存面板，将名称导出为 `.txt` 文件（UTF-8），并自动打开文件位置 |
| **一键选中** | 在 Hierarchy 中选中所有存在空材质问题的 GameObject |

### 结果列表
- 显示总空材质数量 + 各渲染器类型的分类统计
- 每行显示：物体引用 → 红色空材质标识 → 槽位描述（如"材质[0]"、"拖尾材质"）→ 彩色类型标签
- 不同渲染器类型使用不同颜色标签便于区分

## 标签页 2：残留清理

### 功能说明
检测预制体/场景中**未被实际使用**的残留资源引用。例如粒子渲染模式已经不是 Mesh，但 Renderer 上仍残留 Mesh 引用；Trail 模块已禁用，但拖尾材质仍有引用等。

### 检测类型（4大类 9种残留，均可独立开关）

| 类别 | 残留类型 | 说明 |
|------|---------|------|
| **Mesh残留** | 渲染模式不匹配 | 渲染模式非 Mesh 但 Renderer 上仍有 Mesh 引用 |
| | Shape模块不匹配 | Shape 模块禁用或类型不匹配但仍有 Mesh 引用 |
| **Material残留** | Trail模块禁用 | Trail 模块禁用但仍有拖尾材质引用 |
| | 材质文件丢失 | 材质引用的 instanceID 不为0但实际对象为 null（Missing） |
| **贴图残留** | Toggle禁用的模块 | 材质球上被 Toggle 属性禁用的功能模块仍保留贴图 |
| | Shader不需要 | 当前 Shader 不包含该贴图属性，但材质球仍残留贴图数据 |
| **Timeline残留** | 轨道绑定丢失 | PlayableDirector 的轨道绑定对象丢失 |
| | Clip资源丢失 | Timeline Clip 引用的资源丢失 |
| | ExposedRef丢失 | ExposedReference 引用丢失 |

### 操作按钮

| 按钮 | 颜色 | 功能 |
|------|------|------|
| **扫描残留** | 蓝色 | 执行残留扫描 |
| **清理选中 (N)** | 橙色 | 清理所有勾选的残留项（带确认弹窗） |
| **清理全部 (N)** | 红色 | 清理所有残留项（带确认弹窗） |
| **全选** | 默认 | 勾选所有项 |
| **取消全选** | 默认 | 取消所有项的勾选 |

### 结果列表
- 统计面板显示总数 + Mesh/Material/贴图/Timeline 分类数量
- 按 4 个类别折叠分组显示（Foldout）
- 每条记录包含：勾选框 → 物体引用 → 描述文本 → 详细信息 → "清理"按钮 → "定位"按钮

### 特殊说明
- Timeline 绑定丢失仅在**非 Prefab 模式**下检查（Prefab 模式跳过）
- Timeline Clip 资源丢失仅输出 Warning 日志，提示手动处理
- 贴图残留检测通过智能 Toggle 属性匹配（精确/前缀/包含匹配）判断贴图是否被禁用
- 所有清理操作均支持 **Undo 撤销（Ctrl+Z）**

## 标签页 3：缩放修复

### 功能说明
将粒子系统的 `ScalingMode` 从 `Local` 改为 `Hierarchy`，并自动补偿 Scale 保持粒子大小不变。修复后粒子会正确跟随父物体缩放，避免特效大小不一致。

### 两阶段处理逻辑

| 阶段 | 条件 | 操作 |
|------|------|------|
| **阶段1** | Renderer 未启用 | 仅将 ScalingMode 改为 Hierarchy，不调整 Scale |
| **阶段2** | Renderer 已启用 + 当前为 Local 模式 | 改 ScalingMode 为 Hierarchy，并按父级累积缩放计算补偿 Scale |

### Scale 补偿算法
- 从当前 Transform 向上遍历，累乘每级 parent 的 localScale，得到父级累积缩放
- 新 Scale = 原 Scale ÷ 父级累积缩放（各轴独立计算，避免除以零）

### 操作按钮

| 按钮 | 颜色 | 功能 |
|------|------|------|
| **一键修复** | 绿色 | 扫描所有粒子系统并执行两阶段修复 |

### 结果列表
- 统计面板：阶段1处理数、阶段2处理数、跳过数（已是 Hierarchy / 非 Local 模式）、失败数
- 每行显示：物体引用 → 彩色状态标签（✅已修复 / ⏭跳过 / ❌失败）→ Scale 变化信息
- 支持 **Undo 撤销（Ctrl+Z）**

### 跳过条件
- 已是 Hierarchy 模式 → 直接跳过
- 当前为 Shape 等非 Local 模式 → 跳过（不做处理）

## 标签页 4：空粒子清理

### 功能说明
检测粒子系统的 **Renderer** 和 **Sub Emitters** 模块是否开启，两者都未开启则视为"空粒子"，可一键删除。空粒子不产生任何视觉效果，但会增加 DrawCall 和内存开销。

### 检测逻辑

| 状态 | 判定条件 | 处理 |
|------|---------|------|
| ⚠ 空粒子 | Renderer 和 SubEmitters 都未开启 | 标记为可删除 |
| ✅ 有Renderer | Renderer 已启用 | 保留 |
| ✅ 有SubEmitters | SubEmitters 已启用且有有效引用 | 保留 |
| ✅ 两者都有 | Renderer 和 SubEmitters 都启用 | 保留 |

### 自动处理（扫描时自动执行，支持 Undo）
1. **SubEmitters 全空自动关闭**: 如果 SubEmitters 模块开启但所有引用都为 null，自动关闭该模块
2. **SubEmitters 部分空自动清理**: 如果部分引用为 null，自动清理空引用，保留有效引用

### 操作按钮

| 按钮 | 功能 |
|------|------|
| **开始扫描** | 递归扫描所有粒子系统 |
| **全选空粒子 / 取消全选** | 切换所有空粒子的勾选状态 |
| **定位选中** | 在 Hierarchy 中高亮选中所有勾选的空粒子物体 |
| **删除选中 (N)** | 删除勾选的空粒子（红色按钮，带确认弹窗） |

### 结果列表
- 统计面板：总扫描数、空粒子数、有 Renderer 数、有 SubEmitters 数、两者都有数、自动处理统计
- 可勾选"显示安全项"查看所有粒子（默认仅显示空粒子）
- 每行显示：勾选框 → 物体引用 → 彩色状态标签 → Renderer:✓/✗  SubEmit:✓/✗

### 删除行为
- 使用 `Undo.DestroyObjectImmediate` 删除 ParticleSystem 组件
- 同时删除对应的 `ParticleSystemRenderer` 组件
- **保留 GameObject 本身**（仅移除组件）
- 支持 **Undo 撤销（Ctrl+Z）**

## 标签页 5：贴图检查

### 功能说明
扫描指定文件夹下的贴图文件，检查分辨率和文件大小问题，支持一键修复分辨率和自定义压缩。

### 独立扫描设置（不使用共享扫描范围）
- **扫描文件夹**: 文本框输入 Assets/ 路径
- **浏览按钮**: 弹出文件夹选择面板
- **拖拽区域**: 可直接从 Project 面板拖拽文件夹到窗口，自动设置路径并触发扫描；如果拖入的是文件，则取其所在文件夹
- 未输入路径时扫描按钮为灰色禁用状态

### 支持的贴图格式
`.png`、`.jpg`、`.jpeg`、`.tga`、`.psd`、`.bmp`、`.gif`

### 检测规则

| 检测项 | 条件 | 标识 |
|--------|------|------|
| 分辨率不被4整除 | width%4≠0 或 height%4≠0 | 可一键修复（向上取整到最近4的倍数） |
| 超大分辨率 | width>1024 或 height>1024 | 标记为超大贴图 |
| 文件大小超限 | 大于 2MB | 红色高亮，可打开压缩窗口 |

### 操作按钮

| 按钮 | 功能 |
|------|------|
| **开始扫描** | 扫描文件夹中的所有贴图 |
| **强制重导后扫描** | 先对文件夹执行 Force Reimport，然后再扫描 |
| **复制路径** | 将所有问题贴图的资源路径复制到剪贴板 |
| **定位首个** | 在 Project 面板中 Ping 第一个问题贴图 |
| **修复全部** | 批量修复所有分辨率不被4整除的贴图（带确认弹窗） |

### 单条记录操作

| 按钮 | 功能 |
|------|------|
| **定位** | 在 Project 面板中 Ping 该贴图 |
| **复制路径** | 将该贴图的 Asset 路径复制到剪贴板 |
| **修复** | 修复分辨率为4的倍数（向上取整 + 双线性插值缩放） |
| **压缩** | 打开压缩窗口自定义目标分辨率（仅超 2MB 时显示） |

### 结果列表（左右分栏布局）
- **左侧（60%宽度）**: 问题贴图列表，固定 550px 高度
  - 每条显示：序号、路径、问题描述、分辨率、文件大小、建议修改、已修复标识
  - 超 2MB 的条目使用红色字体+红色背景
  - 点击列表项可选中，右侧显示预览
- **右侧（40%宽度）**: 贴图预览面板
  - 显示贴图缩略图（最大 300px，等比缩放，深色背景）
  - 显示路径、分辨率、文件大小、问题描述
  - 点击预览图可定位到 Project 面板（鼠标变为手型光标）
  - 快捷操作：定位到 Project / 修复分辨率 / 压缩贴图

### 贴图压缩窗口（弹出窗口）

点击"压缩"按钮后弹出，提供自定义目标分辨率的压缩功能：

| 功能项 | 说明 |
|--------|------|
| 文件信息 | 显示路径、当前分辨率、当前文件大小（红色） |
| 宽高同步开关 | 修改一边自动同步另一边（默认开启） |
| 宽度/高度输入 | 手动输入 + 预设按钮（64/128/256/512/1024/2048） |
| 快速设置 | 50%（原尺寸一半）、25%（原尺寸四分之一）、保持原比例 |
| 常用预设 | 1024x1024 / 512x512 / 256x256 / 128x128 一键设置 |
| 对齐到4的倍数 | 将当前目标值向上取整到4的倍数 |
| 预估结果 | 显示目标分辨率、压缩比例（百分比）、预估文件大小 |
| 4整除检查 | 如果目标不是4的倍数，显示黄色警告 |
| 备份提示 | 原文件将被备份为 `.backup` 文件 |
| 确认压缩 | 执行压缩并关闭窗口 |

### 特殊功能
- **修复/压缩备份**: 所有修复和压缩操作会自动创建 `.backup` 备份文件
- **双线性插值缩放**: 修复和压缩均使用自实现的 Bilinear 插值算法，保证图像质量
- **Selection 联动**: 在 Project 面板选中贴图时，列表自动跳转到对应条目

## 注意事项

1. 空材质检测、残留清理、缩放修复、空粒子清理的所有修改操作均支持 **Undo（Ctrl+Z）**
2. **贴图检查**的修复/压缩操作**不支持 Undo**，但会自动创建 `.backup` 备份文件
3. 建议操作前备份项目，尤其是批量清理和贴图修复操作
4. Timeline Clip 资源丢失仅标记提示，不会自动删除（需手动处理）
5. Prefab 模式下会跳过 Timeline 绑定丢失检测（因为 Prefab 中绑定状态不同于场景）
6. 扫描大型场景/文件夹可能需要较长时间，工具会显示进度条
7. 贴图检查标签页**必须先指定扫描文件夹**才能开始扫描，避免意外全局扫描
